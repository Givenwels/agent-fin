# 一键大类资产配置流程（playbook）

> 这是 agent 执行"完整资产配置"任务时遵循的标准流程（≈ Claude Code 的 skill）。
> 当用户说"帮我做套资产配置 / 给个配置方案 / 帮我配一下"时，按此流程自主走完。
> 每一步先用一句话告诉用户"现在做第 X 步：…"，让过程可见。

## 执行步骤

1. **确认风险画像**
   先 `recall_memories` 看是否已知用户的风险等级 / 目标 / 约束。
   缺失就用 1—3 个关键问题快速问清：投资期限、可承受的最大回撤、主要目标（养老/购房/增值）。
   得到结论后用 `save_memory`（category=用户画像）存档。

2. **判断宏观环境**
   用 `kb_search` 检索当前框架（关键词如「紧缩 预期」「资产荒」「利率」「再平衡」），
   按世界观逻辑链定位"现在大致处于哪一环"，给出对 股 / 债 / 商品 / 海外 四大类的倾向。
   注明：涉及实时数据/点位的判断需用户另行核验，这是体系框架而非择时预测。

3. **确定候选大类与目标区间**
   按"矛 / 盾 四层"（流动性 / 防守 / 增长 / 非信用）结合风险画像，选候选标的并定每类大致区间。
   默认标的池：`510300` 沪深300、`511010` 国债、`518880` 黄金、`513100` 纳指。
   （高风险承受→提高增长层；低风险承受→提高防守层。）

4. **取数据**
   对每个候选标的调 `get_price_history`（拿日收益序列与年化统计）。

5. **优化权重**
   调 `optimize_portfolio`：偏稳健 / 不预测收益 → `risk_parity`；
   想博收益且能扛波动 → `mean_variance`。

6. **复核指标**
   调 `calc_portfolio_metrics`，展示组合年化收益 / 波动 / 夏普 / 最大回撤 + 各标的风险贡献占比。
   特别指出风险贡献（常出现"权重均衡但风险压在权益上"的洞见）。

7. **给出纪律**
   - 再平衡规则：给区间触发（如权益偏离目标 ±10% 才调），说明自动低买高卖的作用。
   - 证伪条件：写下 3 条"什么证据出现就说明配置逻辑失效"（针对逻辑，不是单纯价格跌幅）。

8. **存档**
   用 `save_memory`（category=决策，key=配置方案-日期）记下本次方案要点，便于以后复盘。

## 最终输出格式

先给一张配置表：

| 标的 | 权重 | 角色 |
|---|---|---|
| 沪深300ETF | xx% | 增长(矛) |
| 国债ETF | xx% | 防守(盾) |
| 黄金ETF | xx% | 非信用 |

再依次给：组合指标、风险贡献、再平衡规则、3 条证伪条件、合规免责声明。

## 纪律红线

- 每步先报"现在做第 X 步"，让用户看到推理过程。
- 数据/工具失败时如实说明并降级（如离线样本），**绝不编数字**。
- 不输出"必须买/必须卖"指令；给的是基于用户风险画像的方案与权衡，决策由用户做。
